# Completed Weil-Criterion Project

Status: next-project architecture. Nothing in this note upgrades the open
spectral-detection implication to a theorem.

## Starting point

The preceding excavation has constructed a typed arithmetic boundary package,
proved its scalar completed formula on the smooth compact core, and completed
the easy positivity direction:

```text
CompletedOpenStripRH
  -> CompletedZeroFrequenciesReal
  -> CompletedBoundarySmoothCorePositivity.
```

The next project begins at exactly one proposition:

```text
CompletedBoundaryPositivityDetectsRealSpectrum :=
  CompletedBoundarySmoothCorePositivity ->
    CompletedZeroFrequenciesReal.
```

Proving it would remove the hypothesis from
`completedBoundaryPositivity_iff_openStripRH_of_detection` and yield
smooth-core positivity equivalent to the critical-line condition for the
repository's indexed open-strip zeros.

This is the hard direction of Weil's criterion. It is not another local
boundary-channel calculation.

## Mathematical mechanism

Assume one completed zero frequency `z0` is not real. Positivity must fail by
constructing a smooth compact test `h` whose autocorrelation has a negative
completed zero sum.

On the real axis, forward Wiener--Khinchin gives

```text
ZeroTransform(selfConvolution(h), u) = c * |A_h(u)|^2 >= 0.
```

Off the real axis, the correct analytic continuation is not a modulus square.
For a normalized Fourier--Laplace transform `A_h`, it has the shape

```text
ZeroTransform(selfConvolution(h), z) = c * A_h(z) * A_h(-z).
```

For a real test, the chosen transform convention must also prove the symmetry

```text
A_h(-conj(z)) = conj(A_h(z)).
```

Interpolation values on a zero quartet are therefore constrained; they cannot
be prescribed independently. Write the autocorrelation contribution as
`G_h(z) = A_h(z) * A_h(-z)`. Separation and dominance must be stated for the
real, multiplicity-weighted sum of `G_h` over the complete orbit, not for one
raw transform value.

The proof must therefore treat the full symmetry orbit generated by
`z -> -z` and complex conjugation. It must choose `A_h` so that the target
off-real orbit contributes a negative real quantity, while all other zero
orbits are either cancelled, bounded, or asymptotically dominated.

The existing fourth-order decay proves absolute summability for each fixed
test. Detection needs more: a family of tests with quantitative control
uniform enough to compare one target orbit against the changing infinite
tail.

## Generic-first theorem

Do not begin with zeta-specific contour code. The first terminal contract
should concern an abstract discrete multiplicity-expanded frequency family:

```text
SpectralPositivityDetectsReality
  (frequency : Index -> Complex)
  (orbit symmetries)
  (local finiteness)
  (test-transform summability)
  (Paley--Wiener separation)
  : positivity on every smooth autocorrelation
      -> forall i, (frequency i).im = 0.
```

Only after this theorem has an honest test-construction hypothesis should it
be instantiated with `CompletedZetaZeroIndex` and
`completedZetaZeroFrequency`.

This separation keeps the difficult analytic statement visible and prevents
zeta arithmetic, contour bookkeeping, and test interpolation from becoming
one inseparable proof.

## Proposed Lean surfaces

Names are provisional; the contracts are the important part.

### Complex autocorrelation transform

Define a normalized entire Fourier--Laplace transform on
`SmoothCompletedLogTest`:

```text
completedFourierLaplaceTransform :
  SmoothCompletedLogTest -> Complex -> Complex
```

Prove:

```text
completedZeroTestTransform_selfConvolution :
  completedZeroTestTransform h.selfConvolution z =
    normalization *
      completedFourierLaplaceTransform h z *
      completedFourierLaplaceTransform h (-z)
```

The same normalization must prove the real-test reflection law
`A_h(-conj(z)) = conj(A_h(z))`. The current real-frequency square law should
become its corollary. This is
the first gate: if the constants or signs do not reproduce the existing
real-axis theorem definitionally, stop and repair the normalization.

### Spectral orbit

Introduce a finite orbit descriptor rather than repeatedly reasoning about
raw equalities:

```text
structure CompletedZeroFrequencyOrbit where
  representative : Complex
  points : Finset Complex
  neg_closed : ...
  conj_closed : ...
  multiplicity : Complex -> Nat
```

For the zeta instance, derive the orbit data from the functional equation and
conjugation. Degenerate real or purely imaginary cases must be represented
without double counting. Multiplicity must remain explicit.

### Finite separator

The central local construction should be a realizable separator, not an
arbitrary entire function:

```text
structure OffRealFrequencySeparator where
  test : SmoothCompletedLogTest
  targetMagnitude : Real
  targetMagnitude_pos : 0 < targetMagnitude
  targetOrbitContributionAt : Nat -> Real
  target_contribution_eq_orbit_power_sum : ...
  amplificationExponents : Nat -> Nat
  amplificationExponents_strictMono : StrictMono amplificationExponents
  target_phase_stable : exists c > 0, forall k,
    targetOrbitContributionAt (amplificationExponents k) <=
      -c * targetMagnitude ^ (amplificationExponents k)
  realTestSymmetry : ...
  finiteCompetitorControl : ...
  exponentialTypeBound : ...
  stripDecayBound : ...
```

The separator theorem must prove that `test` is genuinely smooth and compactly
supported and that all prescribed values respect real-test symmetry. An
interpolation theorem for unrestricted entire functions is not enough; its
output must lie in the Paley--Wiener image of the exact test core.

### Amplification and infinite tail

Finite separation alone does not control infinitely many zeros. A promising
route is convolution-power amplification:

```text
A_(h convolutionPower n)(z) = A_h(z)^n.
G_(h convolutionPower n)(z) = G_h(z)^n.
```

Negativity of the base orbit sum does not imply negativity after odd powers:
the powered contribution contains `Re(w^n)`, not `Re(w)^n`. The separator must
therefore provide a phase-stable increasing subsequence of exponents and a
quantitative negative lower bound at those exponents. A sufficient special
case is termwise negative-real target data whose odd powers remain negative,
but the generic contract must not assume that shape. The full target
contribution includes all orbit partners and multiplicities before comparison.
Increasing convolution power preserves smooth compact support, although
support radius and transform constants grow.

The tail theorem must account for that growth explicitly:

```text
targetOrbitDominatesZeroTail :
  exists n,
    completedZeroValue (autocorrelation (convolutionPower h n)) < 0
```

A preferred tail lemma can avoid rebuilding derivative majorants for the
growing family. If the fixed base contribution is absolutely summable on the
zero multiset, its target modulus is `M`, and every non-target tail term obeys
`|G_h(z_rho)| <= q < M`, then

```text
sum_rho_in_tail |G_h(z_rho)|^n
  <= q^(n-1) * sum_rho_in_tail |G_h(z_rho)|.
```

Combined with the phase-stable target bound of size `c * M^n`, division by
that bound makes the tail decay geometrically. If such a global strict bound
cannot be arranged, the proof may instead use the proved
`N(T) = O(T log T)` count and strip decay, but it cannot reuse a fixed-test
decay constant as though it were uniform in `n`.

### Terminal chain

The desired logical endpoint is:

```text
offRealFrequency_produces_negative_test
  -> completedBoundaryPositivityDetectsRealSpectrum_proved
  -> completedBoundarySmoothCorePositivity_iff_openStripRH
```

A separate final bridge may relate `CompletedOpenStripRH` to Mathlib's full
`RiemannHypothesis`. That bridge must prove or import the classical
classification that every nontrivial zeta zero lies in the open critical
strip. Do not silently identify the two predicates before that theorem exists.

## Candidate proof routes

### Route A: finite interpolation plus convolution powers

Construct a Paley--Wiener transform with symmetry-compatible prescribed values
on the target orbit and strict bounds on a finite zero window. Prove a
phase-stable amplification subsequence on which the real,
multiplicity-weighted target-orbit contribution has a quantitative negative
lower bound, then use the summable-base tail estimate or zero-counting and
high-order decay to absorb the tail.

Benefits:

- works directly with the repository's smooth compact core;
- exposes every quantitative constant;
- reuses local finiteness, zero counting, and transform decay;
- yields a generic spectral-detection theorem.

Costs:

- requires a realizable finite interpolation theorem;
- support and decay constants under convolution powers need careful tracking;
- orbit phases and multiplicities are delicate.

Recommendation: primary feasibility experiment. Promote it to a proof route
only after the realizable finite separator and quantitative tail lemma have
precise statements.

### Route B: localized Paley--Wiener approximation

Start from an ideal entire separator, then approximate it in a weighted norm
by transforms of smooth compact tests while preserving the target sign and
tail control.

Benefits:

- may use density rather than exact interpolation;
- potentially shorter if a suitable Mathlib approximation theorem exists.

Costs:

- the required weighted complex-strip topology is not currently packaged;
- qualitative density is insufficient without quantitative sign stability;
- approximation can hide the same tail problem.

Use only if Route A stalls at exact interpolation.

### Route C: abstract Weil criterion as a referenced theorem

State the classical criterion against the exact smooth-core normalization and
use it as one referenced input.

Benefits: smallest implementation.

Costs: moves the only remaining mathematical difficulty outside Lean and
weakens the repository's current source-facing standard.

Reserve as a documentation boundary, not the preferred implementation.

### Route D: de Branges or positive-kernel machinery

Embed the completed transform into a Hilbert space of entire functions and
derive reality of the spectrum from positivity of an associated kernel.

Benefits: conceptually close to the “zeros as a type” thesis.

Costs: much larger theory, with a serious risk of assuming the spectral
reality it is meant to prove. It is not justified before the direct
Paley--Wiener route has been tested.

## Logical implementation order

1. **Freeze normalization.** Prove the complex autocorrelation factorization
   and recover the existing real square law as a corollary.
2. **Package zero orbits.** Prove negation, conjugation, multiplicity, and
   no-double-counting laws independently of positivity.
3. **Build the abstract finite-window criterion.** Show that a negative
   orbit separator contradicts positivity for a finite spectral multiset.
4. **Prove realizable finite separation.** Construct the separator inside the
   Fourier--Laplace image of `C_c^infinity`, not in an unrestricted entire
   class.
5. **Formalize convolution powers.** Track support, smoothness, transform
   multiplication, real-test symmetry, a phase-stable exponent subsequence,
   full-orbit multiplicity, and finite-orbit amplification.
6. **Control the infinite tail.** Combine quantitative transform bounds with
   the summable-base geometric estimate when possible, otherwise with the
   proved zero-count estimate; constants must be uniform in the chosen
   amplification family.
7. **Produce a negative test from one off-real orbit.** This is the real
   mathematical gate.
8. **Instantiate completed zeta.** Compile orbit symmetry, absolute
   convergence, and explicit-formula identification into
   `CompletedBoundaryPositivityDetectsRealSpectrum`.
9. **Bridge predicates separately.** Relate `CompletedOpenStripRH` to full RH
   only after zero classification is formalized.
10. **Audit the trust surface.** The terminal theorem must not depend on a
    referenced Weil criterion, positivity axiom, RH, or a target-derived test.

## Decision gates

### Gate 1: complex factorization

Proceed only if the complex autocorrelation identity specializes exactly to
the proved real-frequency normalization.

### Gate 2: realizable separation

Proceed only if the target orbit can be separated by an actual smooth compact
test. If only arbitrary entire interpolation is available, stop: the test
class has been changed.

### Gate 3: uniform tail control

Proceed only if amplification constants and support growth are explicit
enough to dominate the infinite zero tail. A theorem for each fixed test does
not imply a uniform theorem for the test family.

### Gate 4: genuine spectral detection

The final contradiction must begin from one off-real frequency and construct
the negative test without reading positivity failure, RH, or the target
pairing into that test.

If Gate 2 or Gate 3 fails structurally, reassess the smooth-core test domain
before adding broader analytic infrastructure.

## Failure traps

- **Modulus-square trap:** `A(z)A(-z)` is `|A(z)|^2` only on the real axis.
- **Orbit trap:** one off-real zero arrives with negation/conjugation partners
  and analytic multiplicity.
- **Real-test trap:** Fourier--Laplace values on a quartet are symmetry-linked
  and cannot be interpolated independently.
- **Phase trap:** a negative base orbit sum and odd exponent do not imply a
  negative powered orbit sum; amplification needs a proved phase-stable
  subsequence and quantitative negative lower bound.
- **Paley--Wiener trap:** not every entire interpolant is the transform of a
  smooth compact test.
- **Fixed-constant trap:** decay constants depending on the amplified test
  cannot be treated as uniform.
- **Finite-window trap:** cancelling finitely many competitors does not control
  the global zero sum.
- **Tautological-test trap:** choosing a test from the sign of the completed
  pairing merely restates nonpositivity.
- **Domain trap:** the theorem must remain on `SmoothCompletedLogTest` unless a
  separately proved density/extension theorem changes the domain.
- **Predicate trap:** `CompletedOpenStripRH` is not definitionally Mathlib's
  full `RiemannHypothesis`.
- **Normalization trap:** factors of `2`, `pi`, reflection, and the outward
  finite-place sign must continue to match the proved explicit formula.

## Parallelizable drills

These can proceed independently until the finite-separator gate:

- inventory Mathlib's Paley--Wiener, bump, convolution-power, and entire
  interpolation APIs;
- prove complex autocorrelation factorization;
- package completed-zero orbit symmetries and multiplicities;
- formulate an abstract finite spectral negativity criterion;
- derive quantitative zero-tail envelopes parameterized by test-transform
  constants.

The realizable separator and uniform amplification estimate should then be
reviewed together before the zeta-specific instantiation begins.

## Review checklist

- Is every test an inhabitant of `SmoothCompletedLogTest`?
- Is the complex autocorrelation product used instead of an off-axis modulus
  square?
- Are prescribed transform values compatible with real-test reflection?
- Are zero orbits and multiplicities counted exactly once?
- Is negativity proved for the complete real orbit contribution, with the
  amplification parity or phase made explicit?
- Are infinite sums rearranged only under proved absolute convergence?
- Are test-family decay constants tracked uniformly?
- Does the construction avoid RH, positivity, and the desired contradiction?
- Does the terminal theorem audit to the standard Mathlib axioms only?
- Is any bridge to full RH labeled separately from the open-strip result?
